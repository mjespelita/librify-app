Version 1.0.3 - Latest

Release Date: April 26, 2025 - https://app.librifyits.com
Release By: Mark Jason Espelita - IT Programmer

System Performance Monitoring

  • New Feature: The developer added Laravel Pulse to the system for enhanced real-time monitoring and diagnostics.

  • Laravel Pulse is a lightweight, self-hosted, real-time application monitoring dashboard. It allows you to visualize and analyze critical activities and events happening inside the application — without relying on any external monitoring services.

  • Access to the Laravel Pulse dashboard is restricted exclusively to the Super Admin role to maintain system security and prevent unauthorized monitoring activities.

  • Monitored Metrics in Laravel Pulse:
    • HTTP Requests: Tracks and displays incoming HTTP requests, including slow requests.

    • Queries: Logs executed database queries with their execution times, highlighting slow queries.

    • Jobs: Monitors queue jobs, including dispatched, processed, and failed jobs.

    • Cache Usage: Displays cache reads, writes, and misses to evaluate cache performance.

    • Exceptions: Captures real-time application exceptions for easier debugging and faster incident response.

    • Queues: Monitors queue health, pending jobs, processing speed, and backlogs.

    • User Sessions: Shows the number of currently active user sessions and live user activities.

    • Scheduled Tasks: Tracks the execution of scheduled (cron) tasks and highlights any failures.

  • Purpose of Adding Laravel Pulse:
    • To enhance real-time visibility into system performance and activities.

    • To help detect slow queries, failed jobs, and exceptions quickly.

    • To allow system administrators and developers to proactively monitor system health and take immediate actions when necessary.

    • To optimize overall performance by providing better insights into traffic, caching, and background processes.

  • The Super Admin can access the Laravel Pulse dashboard by appending "/pulse" to the domain URL. For example https://app.librifyits.com/pulse

Version 1.0.0

Release Date: April 14, 2025 - https://app.librifyits.com
Release By: Mark Jason Espelita - IT Programmer

Inventory Management System

  • Item Management – Includes full CRUD (Create, Read, Update, Delete) operations, tracking of total inventory, quantity in warehouse, quantity on site, damaged items, and associated serial numbers.

  • Site Management – Manage sites with item and technician tracking, including serial numbers and Google Maps integration.

  • Employee Management – Provides full CRUD functionality for managing technicians, employees, and administrators. The system supports three admin roles: Warehouse Admins, who have access limited to warehouse-related operations; Office Admins, who manage office-specific tasks such as task assignments; and Super Admins, who have full access to all features and data. Technicians can access their assigned site information, manage on-site items, and report damages. Additionally, personal information can be stored for each employee, and task reports can be exported to Excel based on a selected date range.

  • On-site and Damage Item Management – Enables detailed tracking and management of items deployed on-site, including the ability to monitor damaged items. All data is organized based on specific sites and includes information about the assigned technicians responsible for each location.

Task Management System

  • Workspace Access Control – Admins (Warehouse, Office, and Super Admins) have the ability to create workspaces. Warehouse Admins can only view and manage workspaces related to warehouse operations, while Office Admins are limited to office-related workspaces. Each admin type is restricted from accessing the other’s workspace to ensure role-specific data security. However, workspaces, projects, and tasks created by the Super Admin are universally visible and accessible to both Warehouse and Office Admins, enabling centralized oversight and collaboration when required.

  • Workspaces can have participants added by Admins or authorized users. Participants are granted access exclusively to the projects and tasks within the specific workspace they are assigned to. This ensures secure, role-based access and allows for streamlined collaboration without exposing data from other unrelated workspaces.

  • Project Management – Inside each workspace, projects can be created to organize tasks. Each project serves as a container for all tasks related to that specific initiative.

  • Task Management – Within each project, tasks are categorized as scheduled or non-scheduled. Scheduled tasks are created once and automatically appear every day at 8 AM through an automated cron job, without requiring human intervention.

  • Comments – Employees, technicians, and admins assigned to a task can collaborate through a real-time comment section. They can also upload and share files, including Word documents, PDFs, images, Excel sheets, and PowerPoint presentations.

  • Task Timer – Employees and admins can start a timer to indicate when they begin working on a task. They also have the ability to pause the timer when needed and mark the task as complete upon finishing, ensuring accurate tracking of work duration and task status.

  • Notification Page – Employees and admins receive notifications for task updates, such as comments, status changes, or file uploads. For example, “Admin John commented on Task #24: Please update the inventory log before 3 PM.”
